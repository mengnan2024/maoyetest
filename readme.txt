 3. 数据传递
 首先完成数据隔离可维护，接下来做更重要的数据传递
 多接口的响应数据与请求参数相互调用

 4. 动态函数
 如当前时间 、 随机生成数据 、数据加密

 5.可配置
 实现一份用例 通过配置文件 实现在不同环境下可以通用执行

 6.可查询日志（可视化）

 7.可视化报告
 清晰 完整 allure

 8.用例驱动
 用例的驱动模式（DDT），存放数据的方式，用例的描述，复用
 从效率的角度 支持并发
